一、使用Powershell

基于Windows和Mac系统命令行的使用差异，可统一使用Powershell，win10系统自带Powershell，其它系统也可安装。

二、打开命令窗口

①默认方式打开：打开”运行”对话框(Win+R)，输入cmd，打开系统默认控制台命令窗口，或者输入Powershell打开Powershell窗口。

②在目标位置打开：在需要打开的文件夹空白处按住Shift，然后右键弹出快捷菜单，可以看到“在此处打开命令行窗口”或者是“在此处打开Powershell窗口”。

三、进入目标文件夹

①更改盘符：默认打开Powershell时会在C盘默认位置，要改变盘符，可使用cd命令，即change directory，如进入E盘，使用cd E:或cd e:，不计大小写，中间空格隔开。

②更改文件夹：cd后跟上要进入的文件夹路径，如cd notes/markdown，会进入到当前位置下的notes文件夹下的markdown文件夹；返回上级目录可使用cd ..，即返回到notes文件夹下。

③也可在打开Powershell窗口后直接把目标文件夹拖入窗口中，这样可直接进入目标文件夹，这个效果和在目标位置打开Powershell窗口一致。

四、创建&删除

①创建文件夹：mkdir，即make directory，例如使用mkdir learn,可在当前位置创建一个名为learn的文件夹。

②删除文件夹：rmdir，即remove directory，例如可使用rmdir learn把刚才创建的learn文件夹删除。

③创建文件：ni，即new item，例如使用ni command.md，可在当前位置创建一个名为command的markdown文件。

④删除文件：rm，即remove，例如可使用rm command.md可把刚才创建的command.md文件删除。

五、一些常用技巧

①上下方向键可查看使用过的命令，可直接回车使用该命令。

②tab键可自动补齐路径、文件夹或文件名称。

③pwd命令(print working directory)可打印当前所在工作目录。

④ls命令(list)可列出当前路径下的内容。

⑤cls命令(clear screen)可对命令窗口进行清屏。

六、关于项目工程化

关于项目工程化的命令行，用得最多的肯定就是npm了，国内可是使用淘宝镜像cnpm，配置方法可去这里查看。

1.npm init可生成package.json文件，用于管理依赖配置。
2.cnpm install 依赖包名可安装对应的依赖，可根据是在开发环境(开发时)还是生产环境(上线后)来加上--save-dev或--save。

①例如安装sass模块：该依赖只需要在开发环境中使用，使用cnpm install sass --save-dev，该配置会在package.json文件的devDependencies节点写入依赖。

②反之如vuex模块在生产环境中也是需要的，所以使用cnpm install vuex --save，该配置会在package.json文件的dependencies节点写入依赖。

③还有需要全局安装的如：webpack的安装，cnpm install webpack -g，需要加上-g尾缀，因为它相当于是给你装上了一个全局的命令。

在实际开发中我们可采用简写方式：install可简写为i，--save-dev可简写为-S，--save可简写为-D。

最后
